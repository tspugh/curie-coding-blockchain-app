# X12 EDI Transaction Sets

A navigation hub for everything in `docs/research/` that touches **X12 EDI** — the ANSI ASC X12 healthcare claim transaction family the project must interoperate with even though state lives on Somnia.

> This file is **navigation, not content**. Every claim about X12 lives in the linked source file; please follow the link before citing.

See also: [[corti]] · [[cda]] · [[README]] (topics index) · [[../../INDEX|docs index]]

---

## Why X12 matters here

The on-chain claim lifecycle records hashes, states, and settlement events — never raw EDI. But the *off-chain* adapters that produce those hashes have to speak the existing X12 stack that providers and payers already use. Several research files therefore ask: where exactly is the package boundary between the on-chain claim struct and the off-chain X12 adapter, and which transaction sets does that boundary need to touch?

The recurring transaction sets in this repo:

| Transaction | Meaning | Direction |
|-------------|---------|-----------|
| **837**     | Claim submission                                 | Provider → Payer |
| **835**     | Remittance / adjudication response               | Payer → Provider |
| **275**     | Patient/claim information attachment (with CDA)  | Provider → Payer |
| **277**     | Claim status / Request for Information (RFAI)    | Payer → Provider |
